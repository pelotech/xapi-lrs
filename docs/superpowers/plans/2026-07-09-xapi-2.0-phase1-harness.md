# xAPI 2.0 Conformance — Phase 1: Harness Upgrade Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the stale conformance-suite fork with the official ADL suite (SHA-pinned), parameterize the harness by xAPI version, harden the vitest wrapper against silent coverage loss, run both batteries in CI, and record the v2_0 red baseline that will drive Phase 2.

**Architecture:** The existing harness (`test/conformance/run-adl-suite.ts` programmatic runner + `adl-conformance.test.ts` vitest wrapper) is kept, adapted to the official suite's runner API. Version selection moves into the runner's `flags` argument (`xapiVersion`), the wrapper gains per-battery section maps and three guards (missing-section, pending, total-floor), and CI grows a `{driver} × {xapi-version}` matrix where 2.0 jobs are non-blocking until Phase 2.

**Tech Stack:** TypeScript, vitest, pnpm (git dependency), the official `adlnet/lrs-conformance-test-suite` (Node/mocha, driven via its `bin/testRunner.js` child-process API), GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-07-09-xapi-2.0-conformance-design.md`

**Verified upstream facts this plan relies on** (checked against commit `5bc232d349c60faded8240da698f195106091638`, master on 2026-07-09):

- The repo's `package.json` name is `adl-lrs-conformance-tests` version `2.0.0-1`, no `prepare`/`postinstall` scripts, same import path as the old npm fork. **However**, it declares `"super-request": "file:./local/super-request"` and `"supertest": "file:./local/supertest"` — pnpm resolves a git dependency's relative `file:` deps against the **consuming project's** root, so a bare git dependency fails to install (`ERR_PNPM_LINKED_PKG_DIR_NOT_FOUND`). Fix: scoped `pnpm.overrides` redirecting those two deps to the `rchadwic` `#ipc` git forks — the exact sources the old npm fork used (verified: its package.json declares `git+https://github.com/rchadwic/{super-request,supertest}.git#ipc`), and the vendored `local/` copies are same-version snapshots of those forks (super-request `0.0.8`, supertest `1.2.0`).
- `TestRunner` constructor signature is unchanged: `(name, owner, flags, lrsSettingsUUID, options, rollupRule)`. `cancel()`, `getCleanRecord()`, the `message` events (`suite start`, `end`), and `summary {total, passed, failed, version}` all still exist and behave as our wrapper expects.
- **Trap:** the child process receives the runner's `flags` object as its option set. `directory` passed in the 5th (`options`) argument is ignored — only `grep`/`optional` are forwarded from `options` — and an unspecified version **silently defaults to the v2_0 battery** (`specConfig.js: defaultVersion: "2.0.0"`). Version selection MUST therefore be `flags.xapiVersion: '1.0.3' | '2.0.0'` (mapped to `v1_0_3`/`v2_0` via `specToFolder`).
- v1_0_3 battery on master: 33 top-level `describe()` sections, all containing the same parenthetical refs (e.g. `(Data 2.4.2)`) our current `SPEC_SECTIONS` uses.
- v2_0 battery: 34 top-level sections. Most reuse parenthetical refs; three do not follow the pattern: the Concurrency section's top-level title is `(4.1.4) Concurrency` (ref leads; bare-sentence describes are nested inside it), `(4.2.7) Additional Requirements for Data Types` also leads with its ref, and `Alternate Request Syntax Requirements` (a single always-passing placeholder test) has no ref.

export type XapiVersion = '1.0.3' | '2.0.0';

export function parseXapiVersion(raw: string | undefined): XapiVersion {
  if (raw === undefined || raw === '1.0.3') return '1.0.3';
  if (raw === '2.0.0') return '2.0.0';
  throw new Error(`Unsupported xAPI conformance version: ${raw} (expected "1.0.3" or "2.0.0")`);
}

- [ ] **Step 3.2: Rewrite `test/conformance/adl-conformance.test.ts`**

Full replacement content:

```typescript
/**
 * ADL xAPI Conformance Test — Per-Section Vitest Wrapper
 *
 * Runs one full ADL battery (selected by CONFORMANCE_XAPI_VERSION, default
 * 1.0.3), then reports results as one Vitest test per spec section, plus
 * three guards that make silent coverage loss impossible:
 *   - a section expected by our map but missing from the log fails
 *   - a pending/cancelled test outside the allowlist fails
 *   - a total test count below the recorded floor fails
 * Guards are skipped when CONFORMANCE_GREP filters the run.
 *
 * Usage:
 *   pnpm test:conformance          # 1.0.3 battery
 *   pnpm test:conformance:2.0     # 2.0 battery
 *   CONFORMANCE_GREP="XAPI-00139" pnpm test:conformance
 */

import { describe, it, expect, beforeAll } from 'vitest';
import {
  runConformanceSuite,
  collectLeafTests,
  type ConformanceSuiteResult,
  type CleanLogNode,
} from './run-adl-suite.ts';
import { parseXapiVersion, PENDING_ALLOWLIST, SPEC_SECTIONS, TOTAL_FLOOR } from './spec-sections.ts';

const XAPI_VERSION = parseXapiVersion(process.env.CONFORMANCE_XAPI_VERSION);
const GREP = process.env.CONFORMANCE_GREP;
const sections = SPEC_SECTIONS[XAPI_VERSION];

let result: ConformanceSuiteResult;

function findSection(match: string): CleanLogNode | undefined {
  return result.log?.tests.find((t) => t.title.includes(match));
}

describe(`ADL Conformance (xAPI ${XAPI_VERSION})`, () => {
  beforeAll(async () => {
    result = await runConformanceSuite({
      xapiVersion: XAPI_VERSION,
      timeout: 600_000,
      grep: GREP,
      onSectionStart: (title) => console.log(`  ▸ ${title}`),
    });
    expect(result.state).toBe('finished');
    expect(result.total).toBeGreaterThan(0);
  }, 660_000);

  it.each(sections)('$label — $match', ({ match, mayBeAbsent }) => {
    const section = findSection(match);
    if (!section) {
      // With a grep filter, unmatched sections simply didn't run.
      if (GREP || mayBeAbsent) return;
      // `return` keeps TypeScript's narrowing happy below (expect.fail throws).
      return expect.fail(
        `Expected suite section matching "${match}" was not reported. ` +
          'Either the battery changed shape (update spec-sections.ts) or the run lost coverage.',
      );
    }

    const leaves = collectLeafTests(section);
    for (const leaf of leaves) {
      if (leaf.status === 'failed') {
        const req = leaf.requirement ? `[${leaf.requirement}] ` : '';
        expect.soft(leaf.status, `${req}${leaf.title}: ${leaf.error}`).toBe('passed');
      }
    }

    const failures = leaves.filter((l) => l.status === 'failed');
    expect(failures).toHaveLength(0);
  });

  it.skipIf(Boolean(GREP))('maps every reported suite section exactly once', () => {
    for (const suite of result.log?.tests ?? []) {
      const matches = sections.filter((s) => suite.title.includes(s.match));
      expect(
        matches.map((m) => m.label),
        `Suite section "${suite.title}" must match exactly one spec-sections.ts entry`,
      ).toHaveLength(1);
    }
  });

  it.skipIf(Boolean(GREP))('has no pending tests outside the allowlist', () => {
    const allowed = new Set(PENDING_ALLOWLIST[XAPI_VERSION].map((a) => a.title));
    const unexpected = result.pendingTests.filter((p) => !allowed.has(p.title));
    for (const p of unexpected) {
      expect.soft(p, `Pending test not in allowlist: [${p.suite}] ${p.title}`).toBeUndefined();
    }
    expect(unexpected).toHaveLength(0);
  });

  it.skipIf(Boolean(GREP))('ran at least the expected number of tests', () => {
    expect(result.total).toBeGreaterThanOrEqual(TOTAL_FLOOR[XAPI_VERSION]);
  });

  it('summary', () => {
    console.log(
      `\nADL Conformance (xAPI ${XAPI_VERSION}): ${result.passing}/${result.total} passing, ` +
        `${result.failing} failing, ${result.pending} pending (${(result.duration / 1000).toFixed(1)}s)`,
    );
  });
});
```
